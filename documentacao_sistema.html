🚀 Introdução ao Sistema

Este sistema foi desenvolvido para automatizar o processamento de relatórios de biblioteca, especificamente para:

�� Relatórios de Empréstimos

Processa dados de empréstimos de livros, organizando por bibliotecas e formatando informações dos usuários.

⏰ Relatórios de Pendências

Gerencia pendências de devolução, organizando por data e biblioteca.

�� Objetivo Principal: Transformar scripts Python em um executável standalone que pode ser usado por usuários finais sem necessidade de instalar Python ou dependências.

🔨 Criação do Executável com PyInstaller

🎯 Foco Principal: Esta seção explica detalhadamente como transformar scripts Python em executáveis standalone usando PyInstaller.

O que é o PyInstaller?

PyInstaller é uma ferramenta que "empacota" aplicações Python em executáveis standalone. Ele analisa seu script Python, encontra todas as dependências e cria um único arquivo executável que pode ser executado em qualquer computador com o mesmo sistema operacional.

Processo de Criação do Executável

1️⃣ Instalação do PyInstaller

def instalar_pyinstaller(): """Instala o PyInstaller se não estiver instalado.""" try: import PyInstaller print("✅ PyInstaller já está instalado") return True except ImportError: print("📦 Instalando PyInstaller...") try: subprocess.check_call([sys.executable, "-m", "pip", "install", "pyinstaller"]) print("✅ PyInstaller instalado com sucesso") return True except subprocess.CalledProcessError: print("❌ Erro ao instalar PyInstaller") return False

2️⃣ Configuração do Comando PyInstaller

def criar_executavel(): """Cria o executável do processador automático.""" print("�� Criando executável...") # Configuração do PyInstaller comando = [ "pyinstaller", "--onefile", # Criar um único arquivo executável "--console", # Mostrar console para logs "--name=ProcessadorBiblioteca", # Nome do executável "--hidden-import=pandas", # Incluir pandas "--hidden-import=openpyxl", # Incluir openpyxl "--hidden-import=xlrd", # Incluir xlrd "processador_simples.py" # Script principal ] try: subprocess.check_call(comando) print("✅ Executável criado com sucesso!") return True except subprocess.CalledProcessError as e: print(f"❌ Erro ao criar executável: {e}") return False

Parâmetros do PyInstaller Explicados

--onefile

Cria um único arquivo executável em vez de uma pasta com múltiplos arquivos. Facilita a distribuição.

--console

Mostra a janela do console durante a execução. Útil para ver logs e mensagens de erro.

--name

Define o nome do arquivo executável gerado.

--hidden-import

Força a inclusão de módulos que o PyInstaller não detecta automaticamente.

⚙️ Processo de Build Completo

Fluxo de Criação do Executável

1️⃣ Preparação

  • Verifica se todos os arquivos necessários existem
  • Instala o PyInstaller automaticamente
  • Prepara o ambiente de build

2️⃣ Análise de Dependências

  • PyInstaller analisa o script Python
  • Identifica todas as importações
  • Detecta bibliotecas necessárias (pandas, openpyxl, xlrd)
  • Resolve dependências transitivas

3️⃣ Criação do Executável

  • Compila o código Python em bytecode
  • Empacota todas as dependências
  • Cria o arquivo executável standalone
  • Gera arquivos temporários (.spec, pasta build/)

4️⃣ Estrutura de Suporte

  • Cria pastas necessárias (Entrada/, Saida/)
  • Gera arquivo README.txt com instruções
  • Cria arquivo .bat para execução fácil
  • Gera instruções de uso completas

5️⃣ Limpeza

  • Remove arquivos temporários do PyInstaller
  • Limpa pasta build/ e arquivo .spec
  • Mantém apenas o executável final

Comando Completo de Build

# Comando PyInstaller completo pyinstaller --onefile --console --name=ProcessadorBiblioteca --hidden-import=pandas --hidden-import=openpyxl --hidden-import=xlrd processador_simples.py # O que cada parte faz: # pyinstaller - Ferramenta principal # --onefile - Um único arquivo executável # --console - Mostrar console # --name=... - Nome do executável # --hidden-import=... - Incluir bibliotecas específicas # processador_simples.py - Script principal

🚀 Como Usar o Sistema

Para Desenvolvedores

Criar o Executável

# 1. Instalar dependências pip install -r requirements.txt # 2. Executar o criador de executável python criar_executavel.py # 3. Ou executar diretamente python instalar_sistema.py

Para Usuários Finais

Uso Simples

  1. Clique duas vezes em Executar_Processador.bat
  2. Coloque arquivos Excel na pasta Entrada
  3. Os resultados aparecerão na pasta Saida
  4. Arquivos processados vão para Entrada/Processados
  5. Arquivos com erro vão para Entrada/Erros

Identificação Automática de Arquivos

def _identificar_tipo(self, arquivo: Path) -> str: """Identifica o tipo do arquivo.""" nome = arquivo.name.lower() if any(palavra in nome for palavra in ["emprestimo", "empréstimo", "loan"]): return "emprestimos" elif any(palavra in nome for palavra in ["pendencia", "pendência", "pending"]): return "pendencias" else: return "desconhecido"
📊 Tipos de Arquivo Aceitos:
  • Empréstimos: arquivos com "emprestimo", "empréstimo", "loan" no nome
  • Pendências: arquivos com "pendencia", "pendência", "pending" no nome
  • Formatos: .xlsx, .xls

🔧 Solução de Problemas

Problemas Comuns na Criação do Executável

❌ Erro: "PyInstaller não encontrado"

Solução: Execute pip install pyinstaller

❌ Erro: "Módulo não encontrado"

Solução: Adicione --hidden-import=nome_do_modulo ao comando PyInstaller

❌ Executável muito grande

Solução: Use --exclude-module para excluir módulos desnecessários

❌ Executável não funciona em outros computadores

Solução: Teste em um computador "limpo" sem Python instalado

Problemas de Uso

✅ Verificar Logs

Os logs são salvos em arquivos com data no nome: processador_20250811.log

✅ Verificar Pastas

Certifique-se de que as pastas Entrada/, Saida/, etc. foram criadas corretamente

Comandos de Debug

# Verificar se PyInstaller está instalado python -c "import PyInstaller; print('OK')" # Testar o script Python diretamente python processador_simples.py # Verificar dependências pip list | grep -E "(pandas|openpyxl|xlrd)" # Criar executável com mais informações de debug pyinstaller --onefile --console --debug=all processador_simples.py
